## 一、问题本质

### 核心命题
传统协作工具的基本单位是「产物 + 评论」。  
AI 时代的协作基本单位应该是「**意图 + 合成**」。

### 识别出的新型协作浪费
A 用 Claude 生成 v1 → B 看完生成 v2 → C 基于 v2 生成 v3  
三个版本互相看不见、合不拢，集体智慧没有沉淀，只留下三份平行草稿。

### 关键区分
这不是"多 agent 协作"问题（OpenClaw 方向），而是**多人协作如何借助 AI 产生更好结果**。主语是人，不是 agent。

## 二、技术问题拆解

### 三个核心子问题

**1. 意图表示（Intent Representation）**
- 自然语言意图需要被解析成可合并的结构
- "首屏要有 hero 图"（结构性） vs "偏冷色风格"（风格性）是不同维度
- 合并前必须先分类，否则合成质量无从保证

**2. 冲突检测与解决（Conflict Resolution）**
- 可叠加冲突："要有 FAQ" + "要有定价" → 直接合并
- 真冲突："暖色调" vs "冷色调" → 需要裁决机制
- 系统不应静默消解冲突，应把决策权还给团队

**3. 合成质量与归因（Synthesis + Attribution）**
- 合成产物必须让每个人认出"我的意图在里面"
- 否则"协作体感"失败 → 退化成"AI 替我们做了一个我们都不认识的东西"
- **意图归因（Attribution）是必须保留的设计**

## 三、系统架构

```
[多人意图输入] → [意图合成引擎] → [产物生成] → [归因展示] → [集体迭代]
```

### 合成引擎：两阶段 Claude 调用

**第一阶段：结构化分析**
```
给定 N 个人的自然语言意图：
1. 提取所有可叠加元素（结构/内容/功能）
2. 识别冲突元素并标注冲突类型
3. 输出统一规格说明（unified spec）
4. 为每个 spec 项标注来源用户
```

**第二阶段：产物生成**  
用 unified spec 作为 prompt，生成实际的 HTML / 代码 / 文档。

## 四、理论地基（来自笔记图片）

### 神经系统隐喻
- 基层（个体意图）= 局部自主，处理高频决策
- 中央（合成引擎）= 整体协调，确保系统稳定
- 关系是**互补的藕合**，不是服从与命令
- 基层活跃是中央高效的前提

### 参与劳动设计 → 缓解异化
- 传统协作：大多数人是"被动执行者"，权力是外在冷漠的官僚结构
- DeepWork 目标：让每个人**从一开始就是生成过程的共同作者**，不是事后评论者
- 产物有我的痕迹 → 归属感 → 协作体感的深层机制

### 分布式决策 + AI 承担噪声
- AI 接管：高频、标准化、重复性判断（噪声处理）
- 人类专注：高价值、高语境的创意意图（不可替代部分）
- 实现"人机功能耦合"

### 对系统设计的具体影响
1. **归因必须可见** — 合成产物里每个人的贡献要有迹可循
2. **冲突不能静默消解** — AI 降噪，但不替人做高语境判断，把决策还给团队

## 六、具体产品设计方案（来自 Claude 另一对话，2026-04-24）

### 6个角色 → 6种意图视角

| 角色 | 典型意图 | 颜色标识 |
|------|---------|---------|
| 设计师 | "首屏要有视觉冲击，风格偏冷色" | 紫 |
| 文案 | "标题要简洁有力，突出核心价值" | 蓝 |
| 程序员 | "要有 API 文档入口，技术参数清晰" | 绿 |
| 产品 | "定价方案突出，FAQ 解答常见疑虑" | 橙 |
| 市场 | "放用户评价，社交证明要强" | 粉 |
| 普通员工 | "整体要看起来专业可信" | 灰 |

> 设计师要简洁 vs 程序员要内容丰富 → 冲突天然存在  
> **冲突可见 + 合成解决** 是最打动评审的 demo 时刻

### 系统三个核心页面

1. **入场页** → 输入名字 + 选角色 + 加入房间
2. **意图板** → 左：输入意图 | 中：实时看所有人的意图 | 右：页面预览
3. **合成结果页** → 带归因高亮的落地页（悬停显示"← 来自设计师"）

## 七、Demo 场景（最小验证）

五名队员同时打开系统，目标：共同生成一个产品落地页。  
各自输入自然语言意图（不分配任务）：
- "希望首屏有 hero 图"
- "强调定价"
- "要有个 FAQ"  
- "整体风格偏冷色"
- "要有 demo 视频位"

系统合成 v1 → 所有人愿意继续迭代 → 支持第二、三轮集体修改。

**成功标准**：每个人能在产物里认出自己的意图，且整体比任何单人版本更完整。
