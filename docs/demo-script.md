# DeepWork 演示脚本

**场景：** 6人团队为一款新产品设计落地页
**时长：** 约 8–12 分钟
**设备：** 每人一台设备（手机/电脑均可），主持人屏幕投影

---

## 演示前准备

1. 确认 `.env.local` 已配置（SUPABASE_URL, SUPABASE_ANON_KEY, ANTHROPIC_API_KEY）
2. `npm run dev` 启动本地服务，或确认 Vercel 部署可访问
3. 主持人提前准备好房间码（页面上点「生成」）
4. 通知 5 位参与者记住自己的角色

---

## 角色分配

| 角色 | 颜色 | 典型关注点 | 建议人选 |
|------|------|-----------|---------|
| 设计师 | 紫 #A855F7 | 首屏视觉、整体风格 | 设计背景的人 |
| 文案 | 蓝 #3B82F6 | 标题、价值主张措辞 | 擅长写作的人 |
| 程序员 | 绿 #22C55E | 技术功能、FAQ技术问题 | 开发背景的人 |
| 产品 | 橙 #F97316 | 定价方案、功能优先级 | PM 背景的人 |
| 市场 | 粉 #EC4899 | 用户评价、社交证明 | 市场背景的人 |
| 普通员工 | 灰 #6B7280 | 整体可信度、新手视角 | 其他任何人 |

## 演示流程

### 第 1 步：进入房间（1分钟）

**主持人说：**
> "我们今天要用 AI 把 6 个人的想法，直接变成一个产品落地页。每个人都是不同角色——设计师、文案、程序员……大家的想法会被 AI 合成进同一个产物，而且你能看到哪个部分是谁提议的。"

**操作：**
1. 主持人打开页面，点「生成」获得房间码（如 `ABC123`）
2. 主持人选自己的角色，输入名字，进入房间
3. 把房间码分享给其他 5 人（口述或屏幕显示）
4. 其他人输入房间码 + 选对应角色 + 输入名字，进入房间
5. Header 会实时显示「N 人在线」，等待 6/6

---

### 第 2 步：提交意图（3–5 分钟）

**主持人说：**
> "现在每个人用自己的角色视角，告诉 AI 你希望这个落地页是什么样的。不需要写代码，不需要画图，就用自然语言描述你的想法。"

**每人操作：**
- 选择一个板块（首屏、定价、功能亮点……）
- 在输入框写下意图，点「提交意图」
- 可以参考页面下方的「示例意图」（点击自动填充）
- 鼓励每人提交 2–3 条，覆盖不同板块

**主持人旁白（观察中间栏实时更新时说）：**
> "你们看到中间这一栏——每个人提交的意图实时出现了，而且颜色对应各自角色。设计师是紫色，程序员是绿色……这是第一次，你能看到团队脑子里在想什么。"

---

### 第 3 步：触发合成（30秒）

**主持人说：**
> "现在 AI 来把所有人的想法整合成一个产品。"

**操作：**
- 任意一人点「合成产物」按钮
- 按钮变灰，显示「合成中...」（正常，约 20–40 秒）
- 所有人页面自动跳转到结果页（无需手动操作）

---

### 第 4 步：查看结果（3分钟）

**主持人说：**
> "这是刚才所有人意图的合成——一个完整的产品落地页。"

**操作演示：**

1. **归因 Overlay**（最关键的演示点）
   - 把鼠标悬停在页面的不同区块上
   - 每个区块会亮起不同颜色的边框，底部出现提示：「[角色] 贡献了这个区块」
   - **主持人说：** "这不只是 AI 生成的，是你们6个人的集体创作。每个区块有它的主要贡献者。"

2. **归因侧栏**
   - 右侧显示每个区块由哪个角色主导
   - 如果有冲突，会显示「冲突解决」说明 AI 如何取舍

3. **下载 HTML**
   - 点「下载 HTML ↓」，得到可直接在浏览器打开的文件
   - **主持人说：** "AI 的输出不是一张 PPT、不是一份会议记录，是可以直接交付的产物。"

---

### 第 5 步：继续迭代（可选，1–2分钟）

**主持人说：**
> "但这只是第一轮。如果你觉得某个地方不够好……"

**操作：**
1. 点「← 继续迭代」回到房间
2. 任意人补充或修改一条意图（比如设计师说「首屏太密集了，要更多留白」）
3. 再次点「合成产物」
4. 结果页面右侧出现「迭代历史」R1 / R2
5. 点击 R1 和 R2 对比变化

**主持人说：**
> "R1 到 R2，你能看到产物如何响应这一条新意图进化。这是传统会议永远做不到的——每一轮迭代都有迹可循，都有归因。"

## 关键话术备用

- **「这和投票有什么区别？」** → 投票是取一个人的方案，这是把所有人的想法融合进同一个产物，没有人被排除。
- **「AI 怎么知道谁的想法更重要？」** → AI 会寻找每个板块贡献最多的意图，归因到对应角色，同时尝试调和冲突。
- **「这能用在真实工作中吗？」** → 这个原型用的是真实的 Supabase + Claude API，今天的演示就是真实运行。

## 如果出现问题

| 问题 | 原因 | 处理 |
|------|------|------|
| 合成卡在「合成中...」超过 60s | API 超时或网络问题 | 刷新页面，重新进入房间，再次触发合成 |
| 跳转到结果页但 iframe 空白 | Claude 返回格式异常 | 检查 Vercel 日志，重试合成 |
| 某人进不去房间 | roomCode 拼写错误 | 确认大写、无空格 |
| 意图提交后中间栏没更新 | Supabase Realtime 连接问题 | 刷新页面 |
